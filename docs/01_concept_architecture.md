# 01. コンセプトとシステム設計（アーキテクチャ）

このドキュメントでは、「Project Antigravity: Discord Sister AI Bot」の全体像と、採用した技術の理由を解説します。

## 1. プロジェクト概要
本プロジェクトは、サバイバルMMOなどのマルチプレイゲームを遊ぶDiscordコミュニティに向けて作られた**「フルボイスの妹分AI」**ボットです。
ただのテキストチャットボットにとどまらず、**Google Gemini** の高度な文脈理解と、**VOICEVOX** の高品質な音声合成を組み合わせることで、実際にボイスチャット（VC）で一緒にゲームをしているかのような没入感を提供します。

### 主な特徴
- **キャラクター性の一貫**: 「お兄ちゃんを慕う妹」という明確なペルソナを持たせ、ゲームの待機時間や作業中の癒やしを提供します。
- **フルボイス応答**: テキストだけでなく、VC上で実際に声を出して反応します。
- **文脈の維持**: 過去の会話やゲーム内の出来事を記憶し、パーソナライズされた対話を実現します。

## 2. なぜこの技術スタックなのか？
本システムは、実用性とモダンな技術学習の両立を目指して以下の技術を選定しています。

- **TypeScript (Node.js)**: 静的型付けにより、Discord.jsの複雑なAPIを安全かつ効率的に扱うため。
- **Google Gemini 1.5 Pro**: 非常に長いコンテキストウィンドウを持ち、過去の膨大な会話履歴を踏まえた自然な妹キャラのロールプレイを安定して行えるため。
- **VOICEVOX**: ローカルおよびコンテナで動作し、無料で高品質な日本語音声合成APIが提供されているため。
- **Google Cloud Run**: サーバーレスでコンテナを実行でき、アクセスがない時のコストを抑えつつ、需要に応じてスケーラブルに運用するため。
- **Cloud Firestore**: スキーマレスで柔軟なデータ保存が可能であり、会話履歴やユーザーごとの「好感度」を高速に読み書きするため。

## 3. システム構成・データフロー
ユーザーのDiscord上での発言は、以下のフローで処理されます。

1. **[入力]**: ユーザーがDiscordで発言（テキストまたはVCコマンド）。
2. **[中継]**: Node.js (Discord.js) アプリケーションがメッセージを受信。
3. **[履歴取得]**: Firestoreから該当ユーザーとの過去の会話履歴や好感度ステータスを取得。
4. **[推論]**: メッセージ、過去の履歴、および「妹」のシステムプロンプトを統合し、Gemini APIへリクエスト。
5. **[音声生成]**: Geminiから返ってきたテキスト応答をVOICEVOXエンジンに送信し、音声データ（WAV）を生成。
6. **[出力]**: 生成された音声をDiscordのVCにストリーミング再生し、テキストチャンネルにも文章で応答を出力。

## 4. キャラクター設定（プロンプト設計）
Geminiに渡すシステムプロンプトには、以下のようなルールを組み込む想定です。
- **一人称/二人称**: 「わたし」 / 「お兄ちゃん」
- **性格**: 明るく健気で、少し甘えん坊。サバイバルゲームの過酷な世界でプレイヤーを応援する癒やし枠。
- **制約事項**: AIやプログラムであることを強調せず、あくまでDiscordサーバーにいる一人の妹として振る舞うこと。

## 5. 今後の展望
- **好感度システム**: Firestoreにユーザーごとの好感度パラメータを蓄積し、数値によって呼び方や声色（VOICEVOXの感情パラメータ）を動的に変化させる。
- **Webダッシュボード**: Reactを用いた管理画面を作成し、好感度の確認やシステムプロンプトの微調整をブラウザから行えるようにする。
